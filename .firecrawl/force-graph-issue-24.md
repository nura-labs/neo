Is there any control available to dictate the framing of the nodes in the graph on load?

The reason I ask is that I'm building a project where you can add a node at any time (disconnected) and boom it appears there in the graph for you to then grab and connect to others or leave on its own. The issue is that these are sometimes incredibly far from the pre-existing cluster of nodes.

For instance if I add 6-7 new disconnected nodes in a row they end up so far away that I have to zoom far out, pan for a while and drag the little buggers back over to the remainder of the graph.

At first I thought this was an effect of the Node radius I had set rather high but it appears this happens at all values of 'r'. I'm wondering if some constraint value that lets you tighten loose nodes closer to the average centroid of all other nodes is possible in an efficient manner.

Thoughts?

NOTE: I am working in 2D mode specifically.

[@GeoffGodwin](https://github.com/GeoffGodwin) the disconnected nodes are being pushed away due to the repelling action from [d3.forceManyBody](https://github.com/d3/d3-force#forceManyBody). This force is balanced with [d3.forceLink](https://github.com/d3/d3-force#forceLink) which attracts nodes connected by a link. Together these two forces form the dynamics of the physics engine. But naturally, disconnected nodes tend to be pushed out as there is no attracting force acting upon them, and pulling them back into the graph.

I can imagine a few approaches to tackle this (from simpler to more complex):

1. You can fix the disconnected nodes into a particular position. At least until they are connected, at which point you can let go of the fixing and let the engine do its job. Fixed nodes are not affected by the physics engine and are set to custom XY coordinates. To fix a node to certain coordinates, simply add `fx` and `fy` attributes with the fixed XY coordinates, to the desired node object.

2. You can configure the `manyBody` force to not repel at large distances, so that the disconnected nodes don't get pushed out beyond a certain point and remain closer. This is just a matter of setting the force's [distanceMax](https://github.com/d3/d3-force#manyBody_distanceMax). Something similar to the following:


```
myGraph.d3Force('charge').distanceMax(30)
```

3. Use your own custom force that acts only on disconnected nodes and pushes them to the center of the canvas, therefore counteracting the effect of `forceManyBody`. A new force can be added using the syntax: `myGraph.d3Force('myCustomForce', function() { /* custom behavior */ })`. This method will probably give you the best results, but it's also more complex to implement.

Good luck, and let me know if you manage to find a useful solution. :)
